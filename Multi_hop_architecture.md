# 2. Generate 250 JSON Streaming Records

## Description
Creates 250 random order records using Python and writes them into ADLS as a JSON file.

This simulates real-world streaming order data generated by applications like e-commerce platforms.

## Command
```python
%python

import json
import random

customers = [
    "Ravi", "Arjun", "Kiran", "Meena", "John",
    "Priya", "Vijay", "Anu", "Rahul", "Sneha",
    "Aakash", "Divya", "Manoj", "Keerthi", "Surya"
]

cities = [
    "Chennai", "Bangalore", "Hyderabad",
    "Mumbai", "Delhi", "Pune",
    "Kolkata", "Coimbatore"
]

products = [
    "Laptop", "Mobile", "Tablet", "Watch",
    "Headphones", "Camera", "TV", "Keyboard"
]

payment_modes = [
    "UPI", "Card", "Cash", "NetBanking"
]

status_list = [
    "SUCCESS", "PENDING", "FAILED"
]

records = []

for i in range(1, 251):

    order = {
        "order_id": i,
        "customer": random.choice(customers),
        "city": random.choice(cities),
        "product": random.choice(products),
        "amount": random.randint(500, 15000),
        "quantity": random.randint(1, 5),
        "payment_mode": random.choice(payment_modes),
        "order_status": random.choice(status_list)
    }

    records.append(json.dumps(order))

data = "\n".join(records)

dbutils.fs.put(
    "abfss://raw-data@databricktesdat.dfs.core.windows.net/tmp/orders/orders_bulk_250.json",
    data,
    True
)

print("250 streaming JSON records created successfully")
```

## Purpose
- Simulates real-time order transactions
- Generates bulk streaming test data

# 4. Read Streaming Data Using Auto Loader (Bronze Layer)

## Description
Uses Databricks Auto Loader with Structured Streaming to continuously ingest JSON files from ADLS.

Auto Loader automatically detects new files and manages schema evolution efficiently.

## Command
```python
bronze_df = (
    spark.readStream
         .format("cloudFiles")
         .option("cloudFiles.format", "json")
         .option(
             "cloudFiles.schemaLocation",
             "abfss://raw-data@databricktesdat.dfs.core.windows.net/checkpoints/schema/orders"
         )
         .load(
             "abfss://raw-data@databricktesdat.dfs.core.windows.net/tmp/orders"
         )
)
```

## Purpose
- Ingests streaming files incrementally
- Supports scalable file-based streaming ingestion

---

# 5. Write Bronze Layer Data

## Description
Stores raw streaming data into the Bronze Delta table.

Bronze layer contains raw, uncleaned source data exactly as received from ingestion.

## Command
```python
(
    bronze_df.writeStream
        .format("delta")
        .option(
            "checkpointLocation",
            "abfss://raw-data@databricktesdat.dfs.core.windows.net/checkpoints/bronze"
        )
        .option("mergeSchema", "true")
        .outputMode("append")
        .table("ext_schema.bronze_orders")
)
```

## Purpose
- Stores raw streaming data
- Provides fault-tolerant ingestion layer

# 7. Transform Data for Silver Layer

## Description
Reads streaming data from Bronze table and applies filtering, cleansing, and enrichment transformations.

Silver layer contains cleaned and business-ready transactional data.

## Command
```python
from pyspark.sql.functions import col, current_timestamp

silver_df = (
    spark.readStream
         .table("ext_schema.bronze_orders")

         .filter(col("amount") > 1000)
         .filter(col("customer").isNotNull())
         .filter(col("order_status") == "SUCCESS")

         .select(
             "order_id",
             "customer",
             "city",
             "product",
             "quantity",
             "payment_mode",
             "amount",
             "order_status"
         )

         .withColumn(
             "total_price",
             col("amount") * col("quantity")
         )

         .withColumn(
             "processed_time",
             current_timestamp()
         )
)
```

## Purpose
- Cleans invalid records
- Applies business transformations and calculations

# 10. Aggregate Data for Gold Layer

## Description
Reads streaming data from Silver table and performs aggregations.

Gold layer contains summarized business metrics used for dashboards and reporting.

## Command
```python
from pyspark.sql.functions import sum, count

gold_df = (
    spark.readStream
         .table("ext_schema.silver_orders")

         .groupBy("city", "product")

         .agg(
             sum("total_price").alias("total_revenue"),
             count("order_id").alias("total_orders")
         )
)
```

## Purpose
- Creates business-level aggregated metrics
- Supports reporting and BI dashboards

---

# 11. Write Gold Layer Data

# 13. Drop Delta Tables

## Description
Deletes Bronze, Silver, and Gold Delta tables from the metastore.

Used during cleanup or before re-running the pipeline.

## Command
```sql

DROP TABLE IF EXISTS ext_schema.silver_orders;
DROP TABLE IF EXISTS ext_schema.gold_sales;
```

## Purpose
- Removes existing Delta tables
- Helps reset the streaming environment

# 14. Remove Checkpoint Files

## Description
Deletes checkpoint folders used by Structured Streaming.

Checkpoint files maintain streaming progress, offsets, and recovery information.

## Command
```python

  "abfss://raw-data@databricktesdat.dfs.core.windows.net/checkpoints/silver",
  True

dbutils.fs.rm(
  "abfss://raw-data@databricktesdat.dfs.core.windows.net/checkpoints/gold",
  True

  "abfss://raw-data@databricktesdat.dfs.core.windows.net/checkpoints/",
```

## Purpose
- Clears streaming state information
- Resets checkpoint tracking for fresh execution

# Overall Architecture Flow

```text
JSON Files (ADLS)
        ↓
Auto Loader
        ↓
Bronze Layer (Raw Data)
        ↓
Silver Layer (Cleaned & Enriched Data)
        ↓
Gold Layer (Aggregated Business Metrics)
        ↓
Dashboards / Reporting / Analytics
```
